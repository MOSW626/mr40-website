# MR40 참가비 결제 운영 가이드

결제 모듈은 스위치식이다. 페이앱(직접 결제)으로 시작하고, 이벤터스로 확정되면
설정 한 줄로 전환된다. 코드 수정은 필요 없다.

## 1. 구조

```
config.js → PAYMENTS: {
  provider: "direct" | "eventus" | "off",
  cardUrl:     "페이앱 결제링크",
  transferUrl: "토스 송금링크",
  eventusUrl:  "이벤터스 행사 페이지",
}
```

- **direct** (기본): 행사 페이지·설문 2단계에 [카드 결제]·[간편 송금] 버튼. URL이 빈 버튼은 "준비 중"으로 표시.
- **eventus**: 두 버튼이 [이벤터스에서 신청·결제하기] 하나로 교체되고, 행사 페이지의 "참가 신청하기"도 이벤터스로 간다.
- **off**: 결제 영역 자체를 숨김.
- 운영 시트(행사정보 탭)의 `payment_provider`, `card_payment`, `transfer_payment`, `eventus_url` 값이 config보다 우선한다. 시트만으로도 전환 가능.

결제 흐름이 노출되는 곳: ① survey.html?id=attend (신청 폼 아래 "2단계 결제" — 제출하면 자동 스크롤) ② event.html 참가비 결제 카드.

## 2. 운영자가 할 일 — 페이앱 (direct 모드)

1. **가입**: payapp.kr → 사업자 회원 — 비영리 사업자등록증, 대표자 신분증, 동아리 통장 사본. 심사 보통 1~3영업일.
2. **링크 만들기**: 결제요청 → 링크결제 →
   - 상품명: `MR 40주년 기념행사 참가비`
   - 금액: **구매자 입력(자유입력)** 으로 설정 ← 중요. 참가비가 인원 구성마다 달라서.
   - 결제수단: 신용카드 + 간편결제 켜기
3. **토스 송금링크**: 토스 앱 → 송금 → 링크로 받기 → 동아리 계좌 지정 (1분, 수수료 0).
4. 두 링크를 CONTENT_TODO 8번에 적거나 운영 시트 행사정보 탭에 넣는다.
5. **환불 규정 문구** 결정 (예: "행사 7일 전까지 전액 환불") → `refund_policy`.

## 3. 운영 루틴 (direct 모드)

- 주 1회: 페이앱 대시보드 결제 내역 + 토스 입금 내역 ↔ 참석 조사 시트 이름 대조.
- 사이트가 결제자에게 "결제자명 = 신청서 이름" 안내를 자동으로 띄우므로 대부분 일치한다.
- 미납자는 단톡방 전체 공지로 리마인드 (개별 독촉보다 잡음이 적다).

## 4. 이벤터스로 전환하기로 했다면

1. 이벤터스에 행사 개설 (신청 문항·티켓 가격 설정).
2. `provider: "eventus"` + `eventusUrl` 기입 (또는 운영 시트에서 `payment_provider`=eventus, `eventus_url` 기입).
3. 끝. 직접결제 버튼은 사라지고 모든 신청·결제 동선이 이벤터스로 모인다.
4. 페이앱 계약은 해지하거나, 동잠 판매 등 다른 용도로 남겨도 된다.

## 5. 주의

- 결제 페이지(페이앱 설정)에 환불 규정을 반드시 표기한다 — 전자상거래 분쟁 예방.
- 페이앱 정산 주기(영업일 기준 수일)를 감안해 행사 직전 결제분은 행사 후 입금될 수 있다.
- 참가비 외 후원금도 같은 링크로 받지 말 것 — 회계 구분이 어려워진다. 후원은 별도 링크/계좌로.
